# Máquina de café com POO. Resposta da pergunta:

No projeto máquina de café nos encontramos a classe Café que herda os atributos da classe Drink neste caso é um exemplo de herança.
Ainda no projeto e na classe café temos a abstração dos valores e sabores bem como as codificações necessárias para que o programa execute os processos sem a necessidade do usuário conhecer a linha de código.
Na classe Drink temos os atributos "sabor e valor" como "private" garantindo assim que esse atributos fiquem protegidos para acesso direto, sendo a unica forma de acessar utilizando os métodos "getters e setters".
Falando sobre polimorfismo, temos a exemplo a classe Drink que podemos utiliza-la para outras classes. No caso do projeto máquina de café, podemos criar por exemplo, uma classe chamada Chá, suco e poderiamos herdar os métodos da classe Drink.
